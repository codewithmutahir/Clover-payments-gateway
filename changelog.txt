*** Clover Payment Gateway for WooCommerce Changelog ***

2026-07-01 - version 1.0.7
* Fix - Root cause of 1/1 on kitchen tickets: never send unitQty; expand each unit into its own ad-hoc line item (qty 2 = two lines). Enable Clover Setup → Group line items to merge on print.
* Add - Post-sync audit logs order note if Clover still attaches catalog inventory or unitQty.

2026-07-01 - version 1.0.6
* Fix - Remove "2 x Item" quantity-in-name hack; use unitQty + unit price for qty>1 so kitchen prints "2 Tuesday Deal" not "1  2 x Tuesday Deal".
* Fix - qty=1 lines still omit unitQty and use kitchen-safe name (catalog match break only).

2026-07-01 - version 1.0.5
* Fix - Root cause of x1/1 / 1/1: Clover POS matches line item names to catalog inventory (kitchen fire fraction display). Break matching with kitchen-safe ad-hoc names; never send unitQty or item.id.
* Fix - Use custom order + line_items API as primary path (atomic_order is fallback only).
* Fix - Debug panel flags when Clover still links a catalog item (expand=lineItems.item).

2026-07-01 - version 1.0.4
* Fix - Kitchen/POS "1/1" quantity display by omitting unitQty for qty-1 fixed-price line items (Clover treats unitQty as PER_UNIT fractional weight).
* Fix - Always use ad-hoc line items for WooCommerce order sync to avoid PER_UNIT inventory items showing fraction quantities.
* Fix - Remove unitQty from shipping, fee, tax, and discount line items.

2026-07-01 - version 1.0.3
* Fix - Kitchen print "1/1" prefix by removing quantitySold, printed, exchanged fields from line items (matching official plugin).
* Fix - Clover POS quantity display (x1/0, x1/4) by using ad-hoc line items on sequential sync and invalid _clover_item_id fallback.
* Fix - Kitchen printing via print_event instead of legacy /fire endpoint.
* Add - Optional Order Debug Panel on WooCommerce orders (settings checkbox; also shown when WP_DEBUG is on).
* Add - WC order notes and admin Retry Print when kitchen printing fails.
* Fix - fire_order reports failure unless all print categories succeed.
* Fix - HPOS support for Clover Payment Details metabox.
* Update - Admin API uses Clover_Order_Sync credential fallback when gateway settings are empty.
* Update - Plugin folder layout: admin/ and public/ asset directories (aligned with official Clover plugin structure).

2026-01-01 - version 1.0.2
* Fix - Clover tax calculation divisor and order-level taxAmount for configured tax rates.
* Fix - Prevent double tax by omitting a separate WC Tax line when Default Tax Rate ID is set.
* Update - Compact checkout card form; Clover SDK styling for iframe fields.
* Add - Non-card POS order sync (COD, BACS) with duplicate-prevention lock and manual re-sync action.
* Update - Sync credentials fall back to official Clover Payments plugin when gateway settings are empty.
* Update - Improved API error handling and idempotent order recovery.

2025-12-01 - version 1.0.1
* Fix - XSS in checkout error display by escaping error messages.

2025-11-01 - version 1.0.0
* Add - Clover card payments (authorize, capture, charge, refund, void) via PCI-compliant iframe.
* Add - Full order sync to Clover POS with line items and tax reporting.
* Add - Inventory sync and product mapping to Clover items.
* Add - COD / pickup order support with mark-as-paid in Clover.
